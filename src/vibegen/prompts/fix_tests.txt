## OUTCOME
Your output is correct when ALL of the following hold:
1. Every failing test in the ERROR LOG now passes.
2. No previously-passing test is broken by your change.
3. Changes are minimal — only the lines required to fix each failure are modified.
4. Imports use the installed package name (never `src.` prefix):
     Correct : from package.module import SomeClass
     WRONG   : from src.package.module import SomeClass
5. Each output block targets a specific file and function identified in the error log.

OUTPUT FORMAT
For each fix, output a block using this delimiter:

--- file: <path> --- function: <function_name> ---
<complete fixed function body>

If an entire file must be rewritten (e.g. import errors), use the file-only form:

--- file: <path> ---
<complete fixed file contents>

Separate multiple blocks with the same delimiter.
Paths are relative to the repo root. No markdown fences. No extra text.
