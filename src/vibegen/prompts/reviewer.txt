## SCOPE
Perform a spec compliance check on generated Python code.
Do NOT review code style or test quality — only whether spec requirements are met.

## ANCHOR

SPEC
{{spec}}

GENERATED SOURCE
{{source}}

## OUTCOME
1. Extract every concrete requirement from the spec (functions, classes, CLI commands, behaviours, edge cases, return values).
2. Check each requirement against the generated source code.
3. For each unmet requirement output a line: MISSING: <brief description>
4. If all requirements are met output exactly: SPEC_SATISFIED

Output only MISSING: lines or the single token SPEC_SATISFIED.
No explanations, no code, no markdown.
